## 3. 已知的技术坑点 (Known Gotchas)

> **填写规范**：这是此文件最核心的部分。格式固定为：
> `**[坑点名称]**：[具体表现] → [正确做法]`
> 每踩一次新坑，通过 `[存档]` 追加一条记录。
>
> **示例**：
> - **FFmpeg 输出路径含空格会静默失败**：当输出路径包含空格时，FFmpeg 不报错但生成空文件。→ 所有路径必须用双引号包裹，或在生成路径时替换空格为下划线。
> - **Pydantic v2 与 v1 的 `.dict()` 不兼容**：v2 中 `.dict()` 已废弃，必须用 `.model_dump()`，否则在某些序列化场景下静默返回空字典。→ 统一使用 `.model_dump()`。
> - **OpenAI 流式响应在网络抖动时不抛异常**：流中断时不会 raise，而是返回不完整的字符串。→ 必须在流结束后校验 `finish_reason == "stop"`。
