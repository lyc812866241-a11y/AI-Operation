# 系统架构与设计模式 (System Patterns)

> **[用途说明]**
> 本文件是项目的"法律与契约"。它定义了系统架构、管线节点（Pipeline Nodes）的边界，以及各个模块之间的交互协议。
> AI 在编写代码前必须遵守这里的每一条规则。如果有新规则，必须通过 `[存档]` 机制更新到这里。

## 1. 系统架构概览 (System Architecture)
[描述系统的高层架构模式，例如是微服务、单体、还是多智能体管线？]

*示例（视频智能体）：*
*系统采用 6 节点（Node 0 -> Node 5）串行状态机模型。每个 Node 是一个独立的沙盒，通过 JSON 契约传递数据，严禁跨节点直接调用内部函数。*

## 2. 核心管线定义 (Pipeline Definition)
[列出系统的核心执行流程和边界。]

*示例（视频智能体）：*
*- **Node 0 (Crawler)**: 资产爬取与原始物料筹备。输入：URL；输出：`raw_data/` 目录。*
*- **Node 1 (Understanding)**: 单集视觉与听觉特征抽取。输入：`raw_data/`；输出：`features.json`。*
*- **Node 3 (Casting)**: 角色分配与逻辑映射。输入：`features.json`；输出：`casting_plan.json`。*

## 3. 核心设计原则 (Design Principles)
[列出不可违反的代码级原则。]

*示例：*
*- **隔离原则 (Isolation Protocol)**: 所有测试产物必须写入专属的 `tests/output/` 沙盒目录，绝对禁止溢出至根目录或 `data/`。*
*- **契约驱动 (Contract-Driven)**: 任何 `skills/` 下的脚本必须有明确的输入输出契约（Schema）。*
