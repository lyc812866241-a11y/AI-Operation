# 任务规格书 (Task Specification)

> **[Lead Agent 专用]** 
> 每次接收到用户的新需求时，Lead Agent 必须先将自然语言需求转化为结构化的技术规格并写入此文件。
> Worker Agent 只能根据此文件中的规格执行代码修改，严禁自由发挥。

## 1. 任务元数据
- **任务名称**: [一句话描述任务]
- **提出时间**: [YYYY-MM-DD]
- **当前状态**: [Draft / Approved / In Progress / Completed]

## 2. 需求解析 (Lead Agent 填写)
- **原始需求**: [用户的自然语言需求摘要]
- **核心目标**: [任务要达成的核心业务目标，如"提升准确率"、"修复崩溃"]
- **验收标准 (DoD)**: 
  - [标准 1: 例如 "通过 tests/test_node3.py 的所有用例"]
  - [标准 2: 例如 "处理单个视频耗时不超过 30s"]

## 3. 技术规格 (Lead Agent 填写)
- **涉及模块**: [受影响的 Node 或目录，如 `src/nodes/node3_casting.py`]
- **核心逻辑修改**:
  - [详细描述要改什么，精确到函数级别]
- **新增依赖**: [如无填 None]
- **潜在风险与影响**: [例如 "可能会改变 Node 4 的输入格式，需要同步修改 Node 4"]

## 4. 执行日志 (Worker Agent 填写)
- **执行人**: Worker Agent (Code Mode)
- **执行步骤**:
  - [ ] 步骤 1: ...
  - [ ] 步骤 2: ...

---
**使用说明**:
1. 用户下达需求。
2. Lead Agent (Architect) 分析需求，填写本文件的 1, 2, 3 部分。
3. 用户回复 "同意"。
4. 切换到 Worker Agent (Code)，根据第 3 部分的规格执行，并勾选第 4 部分的复选框。
5. 完成后执行 `[存档]`。
