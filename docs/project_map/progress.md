# 项目进度与里程碑 (Progress)

> **[用途说明]**
> 本文件是项目的动态进度看板，记录当前状态、已完成的工作和待办事项。
> AI 在每次 `[存档]` 时必须更新此文件，确保进度持久化。
> **此文件是跨会话的项目记忆**——新会话开始时，AI 通过此文件了解项目走到了哪一步。

---

## 1. 整体进度状态 (Overall Status)

> **填写规范**：状态选项：`未开始` / `框架搭建中` / `核心功能开发中` / `测试优化中` / `已上线`

- **状态**：框架搭建中
- **进度估算**：0%（项目接管尚未完成）
- **当前核心焦点**：完成 `[初始化项目]`，建立 project_map 基线

---

## 2. 已完成里程碑 (Completed Milestones)

> **填写规范**：格式固定为 `- [x] [里程碑描述]（完成日期）`
> 每完成一个重要功能或模块，通过 `[存档]` 追加一条记录。
>
> **示例**：
> ```
> - [x] 项目脚手架接入（2025-03-01）
> - [x] IngestNode 完成，支持 mp4/mov/avi 格式标准化（2025-03-05）
> - [x] CutNode 完成，基于 GPT-4o 的自动剪辑点检测（2025-03-12）
> ```

- [ ] 项目接管（`[初始化项目]` 尚未运行）

---

## 3. 当前待办事项 (Current Backlog)

> **填写规范**：按优先级排序，高优先级的放在最前面。
> 每条任务描述必须具体到"可以直接开始做"的粒度，不能写模糊的方向。
>
> **示例**：
> - [ ] **高优先级**：完成 SubtitleNode 的时间戳偏差修复（预计 2 小时）
> - [ ] **高优先级**：为 SubtitleNode 补充单元测试，覆盖中英双语场景
> - [ ] **中优先级**：设计 MusicNode 的接口规范，确认输入输出格式
> - [ ] **低优先级**：优化 IngestNode 的错误提示信息

- [ ] **高优先级**：运行 `[初始化项目]`，完成 project_map 初始化
- [ ] **高优先级**：填写 `systemPatterns.md` 的可用单元清单
- [ ] **中优先级**：确认第一个待开发节点，创建 taskSpec

---

## 4. 已知阻塞问题 (Known Blockers)

> **填写规范**：记录当前阻碍进度的问题。解决后移入已完成里程碑，不保留在此。
>
> **示例**：
> - **Whisper GPU 环境未配置**：本地没有 CUDA 环境，SubtitleNode 只能跑 CPU 模式，处理 10 分钟以上视频会 OOM。需要申请 GPU 机器或改用 API 版 Whisper。
