# Bootstrap Corrections Log

> 记录每次 `[初始化项目]` 过程中 AI 的遗漏、错误判断和用户纠正。
> **下次初始化时，Phase 2 开始前必须先读此文件。**

---

## 使用规则

- 每次 Phase 3 校准时，如果用户指出草稿有遗漏或错误，AI **必须**在此记录，不可跳过
- 格式严格按下方模板填写
- **同一类错误出现 3 次** → 触发升级流程（见下方「升级机制」）
- **30 天内未再出现的错误** → 移入 `## 已解决` 区块

### 升级机制（COUNT >= 3 时）

**执行者：AI（在下一次 [初始化项目] 的 Phase 2 开始前自动执行）**

1. AI 读取本文件，发现 COUNT >= 3 且未标注 `已升级` 的记录
2. AI 将该 LESSON 追加到 `.ai-operation/skills/project-bootstrap/SKILL.md` Phase 2 对应子任务的检查清单中
3. AI 在本文件中将该记录的 COUNT 标注为 `已升级`
4. AI 向用户报告："以下纠正已升级为强制扫描步骤：[LESSON 内容]"
5. **用户确认后**，AI 执行 `[存档]` 保存变更

**为什么 AI 可以改 SKILL.md？**
- SKILL.md 是 AI 的能力协议，AI 有权在经验积累后自我增强
- 但仅限于在 Phase 2 的检查清单中**追加**新检查项，不允许删除或修改已有步骤
- 每次升级必须有用户确认，防止 AI 自行降低标准
