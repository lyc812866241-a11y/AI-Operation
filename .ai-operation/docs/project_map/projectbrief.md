# 核心愿景与业务目标 (Project Brief)

> 北极星文档。架构分歧时以此为准。[STATIC] 极少修改。

## 1. 核心愿景

AI-Operation 是一个 AI Agent 行为治理框架（脚手架），让 AI 编码助手在开发过程中保持纪律——有记忆、有计划、有反思、不偷懒。通过 MCP 工具硬校验 + Git Hook 物理拦截 + 自进化经验库，把"AI 请遵守规则"升级为"AI 不遵守也过不了关"。

## 2. 核心业务指标

| 指标 | 目标值 | 当前值 |
|---|---|---|
| 支持的 IDE 数量 | 4+ | 4（Roo Code / Cursor / Windsurf / Claude Code） |
| MCP 工具数 | 全指令覆盖 | 14 个工具 |
| 测试覆盖 | 全核心机制 | 21 tests passing |
| 存档还原准确率 | 新 AI 读完能接手 | 实战验证中 |

## 3. 目标用户

有 vibe coding 经验的开发者，被 AI 跨窗口失忆、不写计划直接动手、同一个坑反复踩等问题折磨过。不是纯小白，也不是不需要 AI 的资深工程师。

## 4. 明确不做的事

- 不做 AI Agent 执行框架（不是 LangChain/CrewAI 的竞品）
- 不做 IDE 插件（通过 MCP 协议与 IDE 集成，不绑定任何一个）
- 不做 prompt 模板集合（核心价值是强制执行，不是提示词）
