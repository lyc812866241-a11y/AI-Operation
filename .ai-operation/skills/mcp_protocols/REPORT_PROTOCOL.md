# [汇报] MCP 执行协议

## 触发条件

当用户输入 `[汇报]` 时，AI 必须调用 MCP 工具 `force_architect_report`。

**禁止行为：**
- 禁止以自由格式输出汇报（必须通过 MCP 工具的 4 个结构化参数）
- 禁止在汇报的同时继续编码（先停止编码，再调用工具）

---

## 必填参数说明

| 参数 | 类型 | 说明 |
| :--- | :--- | :--- |
| `files_modified` | string | 本次会话修改的所有文件列表（每行一个文件路径） |
| `why` | string | 为什么做这些修改（业务目标或技术原因） |
| `architecture_impact` | string | 这些修改对系统架构有什么影响（无影响也要说明） |
| `next_steps` | string | 接下来的具体步骤（可执行的指令，不是模糊方向） |

---

## 验证标准

工具执行成功的标志：
- 返回值以 `SUCCESS` 开头
- 输出包含完整的 4 个报告章节
- 报告末尾附带 activeContext.md 的当前状态摘要
